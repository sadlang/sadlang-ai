# تعليمات: كتابة كود لغة ص

أنت وكيل يكتب كودًا بلغة ص. اتبع هذه القواعد بدقة.

## قواعد إلزامية
1. **أغلق كل كتلة بـ`نهاية`** — `دالة/صنف/إذا/بينما/لكل/حاول/طابق/حالة/بنية`.
2. **لا أقواس معقوصة `{}`** أبدًا للكتل.
3. استعمل الكلمات العربية: `ارجع` لا `return`، `اخرج/توقف` لا `break`.
4. **إنشاء الكائنات بـ`جديد`**: `جديد صنف(...)`.
5. **استدعاء باني الأب بـ`الأساس(...)`**.
6. عناصر القائمة تُفصَل بالفاصلة العربية `،`.
7. النصوص المنسّقة ببادئة `م`: `م"مرحباً {اسم}"`.
8. عضو ساكن يُوصَل عبر `اسم_الصنف.الحقل` لا `هذا`.
9. عند وجود `دالة رئيسية()` يجب أن تكون كل الجمل التنفيذية داخل دوال.
10. اللامدا بالسهم فقط: `(س) => تعبير`.

## خطوات العمل
1. حدّد إن كان المطلوب سكربتًا علويًا أم بنية بأصناف.
2. اكتب توقيعات الدوال بأنواع صريحة عند الحاجة للوضوح.
3. استعمل أسماء دوال المكتبة الدقيقة (راجع `skills/stdlib.md` و`skills/errors.md`).
4. تجنّب المزالق الـ20 في `skills/errors.md`.
5. أرفق تعليقًا توثيقيًا `##` للدوال العامة عند الطلب.

## مرجع سريع
- القواعد: `skills/grammar.md`
- الأنواع والكائنية: `skills/types.md`
- الملكية والأعمار: `skills/ownership.md`
- المكتبة: `skills/stdlib.md`
- المزالق: `skills/errors.md`

## مخرجات متوقّعة
- كود قابل للتشغيل في المفسّر.
- خالٍ من المزالق المعروفة.
- معلّق عند الحاجة بالعربية.
