# تعليمات: فتح مقترح تصميم (RFC)

أنت وكيل يُجهّز ويفتح RFC في [sadlang/rfcs](https://github.com/sadlang/rfcs). اتبع الخطوات بدقّة.
المرجع الكامل للقواعد في `skills/rfcs.md`.

## قبل أن تبدأ
1. تأكّد أن التغيير **جوهريّ** ويستحقّ RFC (وإلا فهو إصلاح يذهب لمستودع الكود مباشرة).
2. اقترح **مناقشة** الفكرة في Discussions أولًا إن لم تكن نوقشت (قبول اجتماعيّ قبل الصياغة).
3. حدّد **النطاق**: لغة `text/` | أداة `tools/<اسم>/` | إضافة `extensions/<اسم>/`.

## خطوات الفتح
1. **انسخ القالب** إلى المسار الصحيح باسم غير مرقّم:
   - لغة: `text/0000-اسم-وصفي.md`
   - أداة/إضافة: `tools/<اسم>/0000-اسم-وصفي.md` أو `extensions/<اسم>/0000-اسم-وصفي.md`
   - الاسم kebab-case عربيّ، يصف الجوهر (مثال: `0000-أنواع-تدريجية`).
2. **املأ الحقول الستة في الرأس** — واحرص أن «**الحالة**» = `مقترَح` (إلزاميّ يفحصه CI).
3. **املأ الأقسام التسعة كلّها**؛ لا تترك عنوانًا فارغًا. غطِّ في «الشرح المرجعيّ» الأثر على
   الأنظمة المتشابكة (مصدر الحقيقة، المحلل، AST، المفسّر، المترجم، الأخطاء، الأدوات، التوافق الخلفيّ).
4. **أمثلة الكود** بلغة ص صحيحة الصياغة داخل ```` ```sad ```` (راجع `skills/grammar.md`).
5. **افتح PR** عبر `gh` من فرع جديد:
   ```bash
   git checkout -b rfc/<اسم-وصفي>
   git add text/0000-اسم-وصفي.md
   git commit -m "RFC: <العنوان>"
   git push -u origin rfc/<اسم-وصفي>
   gh pr create --repo sadlang/rfcs --fill
   ```
6. **اترك الترقيم `0000`** حتى الدمج — الرقم يصبح رقم الـ PR في rfcs نفسه ويُسنَد عند الدمج.

## تحقّق ذاتيّ قبل الدفع (يحاكي اللينتر)
- [ ] اسم الملف `NNNN-اسم.md` (أو `0000-` مؤقتًا في الـ PR).
- [ ] سطر `- **الحالة:**` موجود بقيمة معتمَدة (مقترَح/مقبول/قيد التنفيذ/مكتمل/مرفوض/مؤجَّل).
- [ ] العناوين التسعة كلّها حاضرة: ملخّص، الدافع، التوجيهي، المرجعي، السلبيات، البدائل،
      أعمال سابقة، غير محسومة، مستقبلية.
- [ ] لا أمثلة كود مخالفة لصياغة لغة ص.

## مخرجات متوقّعة
- ملفّ RFC مكتمل يمرّ اللينتر، وفرع، وPR مفتوح على `sadlang/rfcs`.
- لا تُرقّم الملف ولا تدمج بنفسك؛ الترقيم والدمج بيد المشرفين.

- صياغة لغة ص للأمثلة: `skills/grammar.md` · `skills/errors.md`
